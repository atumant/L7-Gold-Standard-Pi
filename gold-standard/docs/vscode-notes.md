# VS Code notes

## Goal
Use VS Code on the Mac as the primary GUI for:
- GitHub authentication
- source control
- editing the gold-standard project
- optionally working alongside OpenClaw conversations

## Important reality
This Pi session cannot directly install/configure a VS Code extension inside your Mac GUI session unless you do it from the Mac.

## What to use in VS Code
Minimum recommended setup on the Mac:
- built-in Source Control
- GitHub sign-in in VS Code
- optional GitHub Pull Requests and Issues extension

## About an OpenClaw VS Code extension
No verified VS Code extension path has been configured from this Pi session yet. Treat that as a future integration task, not assumed present.

## Recommended operator workflow
1. Authenticate to GitHub in VS Code on the Mac
2. Open the repo in a Mac project folder
3. Use Source Control for normal commits/pushes
4. Keep the Pi as the deployment/runtime target
5. Use OpenClaw chat separately or later integrate a supported tool once validated
